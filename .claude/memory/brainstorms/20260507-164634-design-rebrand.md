# Brainstorm: 디자인 전체 리뉴얼 + 로고 재설계 (명칭 유지)

## 의도

- **산출물**:
  - 새 디자인 토큰 세트 (`globals.css` `:root` + `lib/design-tokens.ts`)
  - 13개 페이지 + 6개 컴포넌트 시각 리뉴얼
  - 새 로고 (lockup + favicon + OG 이미지)
  - `DESIGN.md` 9섹션 포맷 작성 (이후 작업의 단일 진실원)
- **사용자**: 한국 직장인 (연말정산 환급 대상). 1년에 수개월 집중 사용, 민감한 금융·소득 정보 입력
- **트리거 (왜 지금)**: 보안 패치 라운드 종료 시점 — 코드 안정성 확보 후 사용자 신뢰의 다음 축인 **시각적 신뢰**로 이동. Neo-Brutalism의 형광색·도트 배경이 "돈 맡기는 서비스" 인상과 충돌
- **성공 기준**:
  - 1차(정성): 5초 첫인상 테스트에서 "은행/세무 서비스 같다"는 응답 다수
  - 2차(정량): `/calculator` 진입 후 첫 입력까지 평균 시간 단축, 이탈률 감소
  - 3차(브랜드): 로고만 봤을 때 "세금 환급 서비스"임이 추측 가능
- **명칭**: **taxback365 유지** (사용자 결정)

## 제약

### 기술
- Tailwind v4 `@theme inline` config-less 유지 (마이그레이션 비용 회피)
- 토큰 1차 출처: `app/globals.css`, TS 미러: `lib/design-tokens.ts`
- 다크 모드는 현재 미지원 — 이번에 도입할지 여부 결정 필요
- Pretendard 한글 폰트 도입 시 Lexend/Inter 대체 또는 병행 (번들 크기 검토)

### 비즈니스
- 작업 시간 — 13 페이지 × 큰 변경. 단계적 롤아웃 가능 (랜딩·로그인 우선 → 대시보드/계산기 → 관리자)
- 도메인 (taxback365.vercel.app)은 그대로
- 기존 사용자가 있다면 큰 변경에 대한 거부감 — 현재 운영 초기라 영향 작음

### 코드베이스
- 컴포넌트가 6개로 매우 적음 → 페이지 안에 인라인 스타일이 많을 가능성. 리뉴얼 전 공통 컴포넌트 추출 필요할 수 있음
- 색상 하드코딩 금지 규칙(`rules/design.md`) 이미 있음 — 새 토큰만 정의하면 자동 강제

## 추천 + 근거

### 추천: **대안 B (Korean Premium / Toss톤)**

**선택 근거**:
1. **타깃 일치** — 사용자 100%가 한국 직장인. 한국 사용자에게 "신뢰할 수 있는 금융 서비스"의 비주얼 표준은 토스·카카오뱅크 톤이며, 학습 비용 0
2. **Pretendard 단일 폰트** — 한국어 가독성 최상급 + 라이센스 자유 + 단일 폰트로 번들 단순화. 현재 Lexend(영문)+Inter(영문)는 한글 페이지에 부적합
3. **신뢰감 최고** — 사용자가 명시한 "신뢰감 부족" 문제에 가장 직접적 해결
4. **명칭과 호환** — taxback365의 영숫자 명칭에 한글 친화 디자인이 더해지면 "한국 서비스"임이 강해짐

**기각된 대안**:
- **A (글로벌 핀테크)**: 사용자 100%가 한국인인데 글로벌 톤은 거리감 발생. 역으로 현재 Neo-Brutalism이 한국적이지 않다는 문제도 같이 해결 안 됨
- **C (Refined Brutalism)**: 가장 저비용이지만 "신뢰감 부족"을 부분적으로만 해결. 사용자가 명시한 핵심 문제를 절반만 해결하는 안전책 — 전환점에서 적극적 개선이 합리적
- **Z (유지)**: 사용자가 명시적으로 원하지 않음

**B 채택의 위험 완화**:
- "토스 카피캣" 우려 → 차별화 포인트 1개 의도적 도입: ① 한글 자모(`ㅌ`/`ㅂ`)를 활용한 로고 + ② "365" 숫자의 일관된 표기 시스템(예: 모노스페이스 또는 수직 정렬 강조) — 기능적 의미와 묶어 표현
- 다크 모드는 **이번 라운드 out of scope** — 디자인 안정 후 follow-up
